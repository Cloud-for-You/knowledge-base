Virtualizace

Pro virtualizace je využito nástroje multipass, který je možné do MacOS doinstalovat pomocí HomeBrew. Navíc je možnost využívat cloud-init pro úpravu takto vytvářených virtuálních serverů.

Instalace virtualizačního prostředí 

1brew install multipass

Start VM serverů pro běh Kubernetes clusteru s upraveným cloud-init

Pokud používáme multipass virtualizační technologii, je možné využít ansible a plugin pro dynamické inventory společně se statickým inventory. Statické inventory může vypadat následovné pro 3 nodový cluster.

 1[masters]
 2k8s-master01
 3
 4[workers]
 5k8s-worker01
 6k8s-worker02
 7
 8[k8s-hosts:children]
 9masters
10workers

Script pro dynamické inventory využíváme z toho důvodu, aby jsme získali IP adresy VM serverů přímo z multipass a nemuseli je neustále upravovat ve statickém inventory a nebo v DNS.

Playbook pro vytvoření může být následující

 1- name: Prepare VMs for K8S cluster
 2  hosts: localhost
 3  connection: local
 4  tasks:
 5    - name: Create VMs for master nodes
 6      theko2fi.multipass.multipass_vm:
 7        name: "{{ item }}"
 8        cpus: 2
 9        memory: 2G
10        disk: 20G
11        state: started
12        cloud_init: ./files/cloud-config_k8s-hosts.yaml
13      loop: "{{ groups['masters'] }}"
14
15    - name: Create VMs for worker nodes
16      theko2fi.multipass.multipass_vm:
17        name: "{{ item }}"
18        cpus: 2
19        memory: 2G
20        disk: 20G
21        state: started
22        cloud_init: ./files/cloud-config_k8s-hosts.yaml
23      loop: "{{ groups['workers'] }}"

Manuální vytvoření všech VM

1multipass launch lunar --name k8s-master --cpus 2 --disk 20G --memory 2G --cloud-init <path_file>
2multipass launch lunar --name k8s-worker01 --cpus 2 --disk 20G --memory 2G --cloud-init <path_file>
3multipass launch lunar --name k8s-worker02 --cpus 2 --disk 20G --memory 2G --cloud-init <path_file>
